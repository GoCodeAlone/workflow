# Generated Capability Artifacts

This directory contains generated Workflow capability inventory artifacts for
docs and website consumers.

Regenerate from the workflow repository root with:

```sh
GOWORK=off go run ./cmd/wfctl capability ecosystem \
  --repo-root .. \
  --registry data/registry \
  --taxonomy data/capabilities/taxonomy.yaml \
  --format json \
  --output docs/generated/capabilities/ecosystem.json

GOWORK=off go run ./cmd/wfctl capability ecosystem \
  --repo-root .. \
  --registry data/registry \
  --taxonomy data/capabilities/taxonomy.yaml \
  --format md \
  --output docs/generated/capabilities/ecosystem.md

GOWORK=off go run ./cmd/wfctl capability catalog \
  --repo-root .. \
  --registry data/registry \
  --taxonomy data/capabilities/taxonomy.yaml \
  --format json \
  --output docs/generated/capabilities/catalog.json

GOWORK=off go run ./cmd/wfctl capability catalog \
  --repo-root .. \
  --registry data/registry \
  --taxonomy data/capabilities/taxonomy.yaml \
  --format md \
  --output docs/generated/capabilities/catalog.md

GOWORK=off go run ./cmd/wfctl capability crossrefs \
  --repo-root .. \
  --registry data/registry \
  --taxonomy data/capabilities/taxonomy.yaml \
  --format json \
  --output docs/generated/capabilities/crossrefs.json
```

When running from a nested git worktree, pass the workspace root explicitly or
with a relative path that resolves to the directory containing the
`workflow-plugin-*` repositories. The checked-in artifacts should avoid
machine-specific absolute paths.

`ecosystem.json` is the evidence-rich maintainer inventory.
`catalog.json`/`catalog.md` are public docs-facing outputs with raw
uncategorized rows hidden. `crossrefs.json` indexes plugin/provider to
capability links for website navigation and agent lookups.

`schema.json` documents the JSON shape emitted by `wfctl capability ecosystem`
and the shared provider row shape used by catalog output.
Application profiles generated by `wfctl capability app` can include private
service names, plugin names, secret names, and local config paths; review them
before publishing outside the owning team.
