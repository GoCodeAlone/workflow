# Workflow Ecosystem Audit Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a lightweight plan/design audit system that tracks implementation status and external work refs, generates `docs/plans/INDEX.md`, and exposes read-only `wfctl audit plans` and `wfctl audit plugins` commands.

**Architecture:** Add a small internal parser for docs/plans metadata, supporting both YAML frontmatter and legacy docs without frontmatter. Generate a Markdown index from parsed metadata and expose read-only audit commands under a new `wfctl audit` top-level command. Keep plugin auditing read-only at first: inspect local plugin repos, validate manifest shape, and run no network calls.

**Tech Stack:** Go standard library, existing `cmd/wfctl` command style, `gopkg.in/yaml.v3`, existing plugin manifest helpers where possible, Markdown docs in `docs/plans`.

**Step 3: Implement parser**

Implement:

```go
type planDoc struct {
	Path               string
	Title              string
	Status             string
	Area               string
	Owner              string
	ImplementationRefs []planImplementationRef
	ExternalRefs       []string
	Verification       planVerification
	Supersedes          []string
	SupersededBy        []string
	HasFrontmatter     bool
}

type planFinding struct {
	Path     string
	Level    string
	Code     string
	Message  string
}
```

Parser behavior:

- frontmatter is optional for legacy docs
- first Markdown H1 becomes title
- missing frontmatter produces a warning, not an error
- `external_refs` is optional and preserved for index output
- invalid enum values produce errors
- `implemented` without refs produces error
- stale verification date produces warning
