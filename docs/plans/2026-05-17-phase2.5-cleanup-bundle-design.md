## Item 3 detail — regression-guard test pattern

**Naming convention:** match existing gcp test `TestGCPIaCServer_Capabilities_ComputePlanVersionV2` (per cycle-1 I-2). Use `_ComputePlanVersionV2` suffix; do NOT use `_DeclaresV2`. (gcp test stays; aws + azure add equivalent.)

**Constructor pattern:** use plugin's existing no-arg `NewIaCServer()` constructor (verified at `aws/internal/iacserver.go:68`, `azure/internal/iacserver.go:69`, `gcp` — all 3 plugins follow same pattern). Avoids the cross-package `provider.AWSProvider` stub-construction issue cycle-1 I-1 flagged.

Template:

```go
// TestXxxIaCServer_Capabilities_ComputePlanVersionV2 pins the Phase 2
// contract signal: the plugin MUST declare ComputePlanVersion="v2" so
// wfctl routes via wfctlhelpers.ApplyPlanWithHooks (v2 dispatch).
// A silent regression that drops the literal would degrade to v1
// dispatch via wfctlhelpers.DispatchVersionFor's empty-default,
// breaking the Phase 2 hard-cutover contract per ADR 0024 + ADR 0040.
// Per workflow#640 + #695 Phase 2 + 2.5 cascade closeout.
func TestAWSIaCServer_Capabilities_ComputePlanVersionV2(t *testing.T) {
    s := NewIaCServer() // no-arg constructor; matches gcp's existing test
    resp, err := s.Capabilities(context.Background(), &pb.CapabilitiesRequest{})
    if err != nil { t.Fatalf("Capabilities: %v", err) }
    if got := resp.GetComputePlanVersion(); got != "v2" {
        t.Errorf("CapabilitiesResponse.ComputePlanVersion = %q; want %q (v2 dispatch opt-in lost)", got, "v2")
    }
}
```

Per-plugin adaptations:
- **aws**: receiver `awsIaCServer`, file `workflow-plugin-aws/internal/iacserver_test.go` (extend existing file; no new file per cycle-1 I-6)
- **azure**: receiver `azureIaCServer`, file `workflow-plugin-azure/internal/iacserver_test.go` (currently has no Capabilities-related test; verified via `gh api .../azure/internal/iacserver_test.go` grep returned empty)

Test file ALWAYS lives in `internal/iacserver_test.go`. No "or new file" knob (cycle-1 I-6 fix).

## Item 2 detail — DOProvider.Apply deletion

### Current state (verified)

`internal/provider.go:274-310` defines `DOProvider.Apply` as a v1 wrapper around `wfctlhelpers.ApplyPlan` + post-loop deferred-flush iteration (lines 295-307). Per Phase 2.5 (workflow#695), DO now declares `ComputePlanVersion="v2"` → wfctl routes via `wfctlhelpers.ApplyPlanWithHooks` directly + calls plugin's `IaCProviderFinalizer.FinalizeApply` RPC → `doIaCServer.FinalizeApply` inlines the same per-driver deferred-flush iteration.

`DOProvider.Apply` is now **unreachable** under v2 dispatch. No in-process embedders verified (grep -rn DOProvider --include="*.go"`).

### Critical pre-decision: is `Apply` still a required method on `interfaces.IaCProvider`?

If yes → cannot delete the method body; replace with error stub. If no → clean deletion possible. **MUST verify at plan-write time** via:
```bash
grep -n "Apply(" /Users/jon/workspace/workflow/interfaces/iac.go
```

For this design, **proceed assuming Apply IS still on the IaCProvider interface** (conservative — Phase 2.5 didn't touch the interface).

### Deletion scope (assuming interface keeps Apply)

1. **Replace `DOProvider.Apply` method body** at `internal/provider.go:274-310` with sentinel-error stub (cycle-1 minor — use sentinel error variable for `errors.Is` traversability):
   ```go
   // ErrApplyV1Removed is the sentinel returned by DOProvider.Apply
   // when called post-Phase-3 cleanup. Callers can `errors.Is` against
   // it for diagnostic classification.
   var ErrApplyV1Removed = errors.New("DOProvider.Apply: v1 dispatch removed since DO v1.4.0 (workflow#695 Phase 2.5 + Phase 3); upgrade wfctl to v0.55.0+ for v2 dispatch via wfctlhelpers.ApplyPlanWithHooks + IaCProviderFinalizer.FinalizeApply")

   func (p *DOProvider) Apply(ctx context.Context, plan *interfaces.IaCPlan) (*interfaces.ApplyResult, error) {
       // v2 dispatch only since DO v1.3.0 (workflow#695 Phase 2.5).
       // Removed in DO v1.4.0 (Phase 3 cleanup). wfctl bypasses this
       // method when ComputePlanVersion="v2" is declared in Capabilities.
       // Deferred-flush behavior moved to doIaCServer.FinalizeApply via
       // IaCProviderFinalizer RPC. Reaching this method indicates a
       // misconfigured caller (in-process embedder using a pre-Phase-2.5
       // wfctl tag, OR gRPC consumer that opted out of v2 dispatch).
       return nil, ErrApplyV1Removed
   }
   ```
   Net: ~30 lines removed, ~12 added.

   **Architectural note on interface segregation** (cycle-1 C-5 review consideration): the reviewer suggested splitting `Apply` off `IaCProvider` into a deprecated `LegacyApplier` interface for compile-time elimination of the runtime-failure surface. **Deferred to a separate workflow-side refactor design.** Reasons:
   - Interface change affects all 4 IaC plugins simultaneously (aws/gcp/azure also implement Apply via similar wrappers); coordinated 5-repo cascade like Phase 2 would be required.
   - Bundle scope is mechanical cleanup; introducing an interface refactor mid-bundle violates the scope discipline that makes the bundle reviewable.
   - DO is the first plugin to need this; the right refactor pattern emerges from observing aws/gcp/azure's wrappers (do they ALSO have post-loop hooks like deferred-flush? Phase 1 inventory said no).
   - File followup tracking issue post-bundle.

   The sentinel-stub approach is the conservative ADR-0024-compliant choice for this bundle: ADR 0024 mandates "compile-time safety" for **new contract surfaces** (the v2 RPC contract), not retroactive removal of v1 surfaces. The stub preserves the IaCProvider interface contract (Apply still implemented), surfaces misconfiguration with an actionable error, and stays inside the bundle's mechanical-cleanup scope.

2. **Relocate `deferredUpdater` named interface** (cycle-1 I-3 fix — keep as named type for readability + Effective-Go "name interfaces by method" pattern). Move from `internal/provider.go:228-241` to `internal/iacserver.go` (where the sole consumer `FinalizeApply` at line 217 lives). The named type preserves Go-level type clarity vs an anonymous inlined interface.

3. **Delete `internal/provider_deferred_test.go`** entirely. `internal/iacserver_finalize_test.go` (Phase 2.5 Task 8) covers the same regression invariant via v2 dispatch.

4. **Collapse godoc rationale** at `internal/provider.go:228-273` — multi-paragraph design rationale leading up to Apply reduces to a one-paragraph note explaining the stub.

5. **No interface assertion changes** at `internal/provider.go:41-43` — `*DOProvider` still satisfies `interfaces.IaCProvider.Apply` (just returns error).
