## Item 3 detail — regression-guard test pattern

Template (DO's existing test at `internal/iacserver_finalize_test.go:192` is the canonical reference). For aws/gcp/azure, copy-adapt:

```go
// TestXxxIaCServer_Capabilities_DeclaresV2 locks the load-bearing
// ComputePlanVersion="v2" opt-in literal in the Capabilities RPC
// return statement. Without this regression-guard, a future
// refactor of the return literal could silently revert to v1
// dispatch (empty ComputePlanVersion → wfctlhelpers.DispatchVersionFor
// defaults to v1) with no signal until production deploy fails.
// Per workflow#640 + #695 Phase 2 + 2.5 cascade.
func TestAWSIaCServer_Capabilities_DeclaresV2(t *testing.T) {
    s := newAWSIaCServer(<stub-provider>)
    caps, err := s.Capabilities(context.Background(), &pb.CapabilitiesRequest{})
    if err != nil { t.Fatalf("Capabilities: %v", err) }
    if got := caps.GetComputePlanVersion(); got != "v2" {
        t.Errorf("CapabilitiesResponse.ComputePlanVersion = %q; want %q (v2 dispatch opt-in lost)", got, "v2")
    }
}
```

Per-plugin adaptations:
- **aws**: receiver `awsIaCServer`, constructor `newAWSIaCServer(provider *AWSProvider)`
- **gcp**: receiver `gcpIaCServer`, constructor `newGCPIaCServer(provider *GCPProvider)`
- **azure**: receiver `azureIaCServer`, constructor `newAzureIaCServer(provider *AzureProvider)`

Test file lives in `internal/iacserver_test.go` (or new `internal/capabilities_test.go` if test file is crowded). Constructor stub mirrors DO's pattern.

## Item 2 detail — DOProvider.Apply deletion

### Current state (verified)

`internal/provider.go:274-310` defines `DOProvider.Apply` as a v1 wrapper around `wfctlhelpers.ApplyPlan` + post-loop deferred-flush iteration (lines 295-307). Per Phase 2.5 (workflow#695), DO now declares `ComputePlanVersion="v2"` → wfctl routes via `wfctlhelpers.ApplyPlanWithHooks` directly + calls plugin's `IaCProviderFinalizer.FinalizeApply` RPC → `doIaCServer.FinalizeApply` inlines the same per-driver deferred-flush iteration.

`DOProvider.Apply` is now **unreachable** under v2 dispatch. No in-process embedders verified (grep -rn DOProvider --include="*.go"`).

### Critical pre-decision: is `Apply` still a required method on `interfaces.IaCProvider`?

If yes → cannot delete the method body; replace with error stub. If no → clean deletion possible. **MUST verify at plan-write time** via:
```bash
grep -n "Apply(" /Users/jon/workspace/workflow/interfaces/iac.go
```

For this design, **proceed assuming Apply IS still on the IaCProvider interface** (conservative — Phase 2.5 didn't touch the interface).

### Deletion scope (assuming interface keeps Apply)

1. **Replace `DOProvider.Apply` method body** at `internal/provider.go:274-310` with explicit-error stub:
   ```go
   func (p *DOProvider) Apply(ctx context.Context, plan *interfaces.IaCPlan) (*interfaces.ApplyResult, error) {
       // v2 dispatch only since DO v1.3.0 (workflow#695 Phase 2.5).
       // wfctl bypasses this method when ComputePlanVersion="v2" is
       // declared in Capabilities. Deferred-flush behavior moved to
       // doIaCServer.FinalizeApply via IaCProviderFinalizer RPC.
       //
       // Reaching this method indicates a misconfigured caller:
       //   - in-process embedder using a pre-Phase-2.5 wfctl tag, OR
       //   - gRPC consumer that opted out of v2 dispatch.
       // Return an actionable error rather than running the dead v1 path.
       return nil, fmt.Errorf("DOProvider.Apply: v1 dispatch removed since DO v1.3.0 (workflow#695 Phase 2.5); upgrade wfctl to v0.55.0+ for v2 dispatch via wfctlhelpers.ApplyPlanWithHooks + IaCProviderFinalizer.FinalizeApply")
   }
   ```
   Net: ~30 lines removed, ~8 added.

2. **Delete `deferredUpdater` interface declaration** at `internal/provider.go:228-241`. Inline equivalent anonymous interface at the FinalizeApply call site in `internal/iacserver.go::FinalizeApply` (replace `du, ok := d.(deferredUpdater)` with `du, ok := d.(interface{ HasDeferredUpdates() bool; FlushDeferredUpdates(context.Context) error })`).

3. **Delete `internal/provider_deferred_test.go`** entirely. `internal/iacserver_finalize_test.go` (Phase 2.5 Task 8) covers the same regression invariant via v2 dispatch.

4. **Collapse godoc rationale** at `internal/provider.go:228-273` — multi-paragraph design rationale leading up to Apply reduces to a one-paragraph note explaining the stub.

5. **No interface assertion changes** at `internal/provider.go:41-43` — `*DOProvider` still satisfies `interfaces.IaCProvider.Apply` (just returns error).

### Add regression test for stub

New test in `internal/provider_apply_stub_test.go`:
```go
func TestDOProvider_Apply_ReturnsRemovedError(t *testing.T) {
    p := &DOProvider{}
    _, err := p.Apply(context.Background(), &interfaces.IaCPlan{})
    if err == nil { t.Fatal("expected DOProvider.Apply to return error") }
    if !strings.Contains(err.Error(), "v1 dispatch removed") {
        t.Errorf("error message missing 'v1 dispatch removed' marker: %v", err)
    }
}
```

## DO tag bump: v1.4.0 (minor)

Rationale:
- Behavioral change (Apply now returns error instead of succeeding) is observable to any caller.
- Even though no real-world callers exist, semver convention treats this as a behavioral break, not bugfix.
- v1.4.0 signals to downstream consumers (BMW + future) that something changed beyond pure cleanup.
- Patch (v1.3.1) defensible if argued as "fixing dead code" but inconsistent with DO's history (minor bumps for behavioral changes since v1.0.0).
